# channeltalk-cs-onboarding-delta 퀵스타트 안내

**읽기 전용 온보딩 스킬이다. 파일을 수정하지 않고, 판정·수치를 즉석에서 창작하지 않는다. 기대 결과(어긋남·차이 건수, 어떤 슬롯이 걸리는지)를 채팅에 미리 말하지 않는다** — 어떤 어긋남이 몇 건 나오는지는 시연 실행이 스스로 보여줘야 하며, 미리 말하면 이후 차이점 추출이 답을 베낀 것처럼 보이게 된다(기대 결과 문서는 `sample/EXPECTED.md`·`docs/quickstart.html` — 심사·검증용).

## 절차

### 1) 문서 위치 확인
이 SKILL.md의 **두 단계 상위**(`../..`)가 플러그인 루트다. 거기서 `docs/quickstart.html`이 있는지 확인한다.
(예: 이 파일이 `<루트>/skills/channeltalk-quickstart/SKILL.md`이면 문서는 `<루트>/docs/quickstart.html`.)

### 2) OS 브라우저로 열기 시도
플랫폼에 맞는 명령으로 문서를 연다. 실패하거나 헤드리스 환경이면 조용히 건너뛴다(에러로 중단하지 않음).
- macOS: `open "<루트>/docs/quickstart.html"`
- Windows: `Start-Process "<루트>\docs\quickstart.html"` (또는 `cmd /c start "" "<루트>\docs\quickstart.html"`)
- Linux: `xdg-open "<루트>/docs/quickstart.html"`

### 3) 채팅 출력 — 문서가 열렸으면 짧게, 못 열었을 때만 요약

**2)에서 브라우저 열기에 성공한 경우(기본)**: 문서가 곧 온보딩이다. **HTML에 이미 있는 내용(기대 결과 표·설치 명령·근거 링크 등)을 채팅에 다시 옮기지 않는다.** 시연 제안은 **파일명이 아니라 고객사 이야기로** 한다. 채팅 출력은 아래 3줄을 넘지 않는다:
1. 소개 문서를 브라우저에 띄웠다는 확인 한 줄
2. 용도 한 줄: 새 고객사의 흩어진 CS 문서를 이커머스 CS 표준 매뉴얼과 대조해 **표준과 다른 것·빠진 것·서로 어긋난 안내·신규 주제**만 근거 인용과 함께 찾아, AI 상담원 '알프' 온보딩의 지식변환 손작업을 가속하는 도구
3. 시연 제안 한 줄: "새 고객사를 받았다고 해볼까요? 시연은 파일 두 개를 씁니다 — 기준이 되는 **채널톡 표준 CS 매뉴얼**과, 신규 고객사 '아크메샵'의 고객센터 운영 문서(반품·환불 정책·FAQ·상품 안내가 한 파일). 표준 매뉴얼 20슬롯에 고객사 문서를 대입해 이 회사만 다른 점·빠진 항목·서로 어긋난 안내를 찾아 안내해 볼까요?" (**어떤 어긋남이 몇 건 나오는지 결과를 미리 말하지 않는다** — 결과는 실행이 보여준다.)

**브라우저 열기에 실패한 경우(헤드리스/무GUI)에만** `docs/quickstart.html`을 직접 읽어 채팅에 요약한다:
- **용도** 1~2줄: 고객사 문서(정책·FAQ·상품)를 이커머스 CS 표준 매뉴얼과 대조해 차이점만 근거와 함께 리포트하는 CS 온보딩 차이점 도우미. 공통 문의는 표준이 커버하니 회사별 차이점만 사람이 검수한다.
- **돌리면 세 가지**: ① 차이점 리포트(서로 어긋난 안내·다른 정책을 `파일:라인 "원문"`과 함께) ② 고객사에 물어볼 질문 목록(문서에 근거 없는 슬롯) ③ 신규 주제(표준에 없는 이 회사 고유 주제).
- **사용법 3경로**: ① Codex 앱 "사용해 보기" ② 채팅 복붙 프롬프트 ③ CLI 설치+실행.
- 마지막에 시연 제안 한 줄 — 문서가 안 열렸으므로 시연용 두 파일(기준 `표준-CS-매뉴얼.md` + 고객사 문서 `acme-shop.md`)을 채팅에서 직접 소개하며, 두 파일을 함께 첨부하도록 제안한다. (기대 결과 예고 금지.)

### 4) 시연 수락 시 — 파일은 사용자가 직접 첨부하게
**바로 번들 샘플을 로드해 돌리지 않는다** — 그러면 사용자에게는 실제 추출과 미리 완성된 골든 호출이 구분되지 않는다. 대신:
1. macOS면 `open -R "<루트>/sample/demo/acme-shop.md"`로 시연 폴더(`sample/demo/`)를 Finder 창에서 연다 — 그 창에 **두 파일**이 함께 보인다: 기준 `표준-CS-매뉴얼.md`와 고객사 문서 `acme-shop.md`(파일을 지정해야 그 파일이 담긴 폴더 창이 실제로 열린다; 다른 OS면 절대 경로를 채팅에 제시).
2. 채팅으로 안내: "Finder 창에 시연용 파일 두 개를 열어 두었습니다 — ①채널톡 표준 CS 매뉴얼(`표준-CS-매뉴얼.md`, **기준**) ②신규 고객사 아크메샵의 문서(`acme-shop.md`). **두 파일을 함께 이 채팅에 끌어다 놓아 주세요.** 표준 매뉴얼에 고객사 문서를 대입해 다른 정책·빠진 항목·서로 어긋난 안내를 찾아 안내해 드립니다."
3. 사용자가 파일을 첨부하면 **onboarding-delta** 스킬로 넘긴다 — 그 스킬이 §0.7대로 표준 매뉴얼을 기준으로 삼고 고객사 문서의 차이점을 추출한다(이 스킬은 추출을 직접 수행하지 않는다).
4. 사용자가 첨부 없이 "그냥 돌려줘"라고 하면 그때만 번들 샘플로 진행하고, 응답 서두에 **"번들 샘플 시연"**임을 밝힌다.

## 주의
- 이 스킬은 안내만 한다. 차이점 추출·분류·리포트 생성은 onboarding-delta 스킬과 `engine/cli.py`가 수행한다.
- 기대 결과(어긋남·차이 건수, 어떤 슬롯이 걸리는지)를 미리 말하거나 없는 수치를 지어내지 않는다 — 결과는 추출·분류 실행이 보여주고, `sample/EXPECTED.md`는 심사·검증용으로만 존재한다.

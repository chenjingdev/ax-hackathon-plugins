# musinsa-quickstart — 퀵스타트 온보딩 안내

이 스킬은 **읽기 전용 안내 스킬**이다. 어떤 파일도 수정하지 않는다. 판정·수치를 즉석에서 창작하지 않는다. **기대 판정 값을 채팅에 미리 말하지 않는다** — 어떤 샘플이 무슨 판정을 받아야 하는지는 시연 실행이 스스로 보여줘야 하며, 미리 말하면 이후 검수가 답을 베낀 것처럼 보이게 된다(기대 결과 문서는 `sample/EXPECTED.md` — 심사·검증용).

## 절차

### 1. 플러그인 루트 찾기
이 `SKILL.md`가 있는 디렉토리의 **두 단계 상위(`../..`)**가 플러그인 루트다(`skills/musinsa-quickstart/SKILL.md` → 루트). 루트에서 `docs/quickstart.html`이 존재하는지 확인한다. 없으면 3)의 요약만 수행한다.

### 2. 사람 사용자를 위해 브라우저로 열기 시도
OS에 맞춰 `docs/quickstart.html`을 기본 브라우저로 연다(사람이 보면 가장 좋은 온보딩):
- macOS: `open <루트>/docs/quickstart.html`
- Windows(PowerShell): `Start-Process <루트>\docs\quickstart.html` — 또는 `cmd /c start "" <루트>\docs\quickstart.html`
- Linux: `xdg-open <루트>/docs/quickstart.html`

문서 하나만 연다(시연 상품의 등록 화면 캡처는 문서 안에 실려 있다 — 별도 창으로 띄우지 않는다). 명령이 실패하거나 헤드리스/무GUI 환경이면 조용히 건너뛰고 3)으로 진행한다(오류로 중단하지 않는다).

### 3. 채팅 출력 — 문서가 열렸으면 짧게, 못 열었을 때만 요약

**2)에서 브라우저 열기에 성공한 경우(기본)**: 문서가 곧 온보딩이다. **HTML에 이미 있는 내용(샘플 표·설치 명령·경제 효과 등)을 채팅에 다시 옮기지 않는다.** 시연 제안은 **파일명이 아니라 상품 이야기로** 한다(사용자에게 `input.product.json`은 블랙박스다). 채팅 출력은 아래 3줄을 넘지 않는다:
1. 소개 문서를 브라우저에 띄웠다는 확인 한 줄
2. 용도 한 줄: 무신사 상품 표시정보(혼용률·원산지·친환경 표현·시험성적서) 컴플라이언스를 검수·재판정하는 도구
3. 시연 제안 한 줄: "문서에 등록 화면이 실린 '친환경 에코레더 무스탕 자켓'(690,000원) — 이 상품이 검수에 걸리는지 확인해볼까요?" (**위반 내용·기대 판정을 미리 말하지 않는다** — 결과는 검수가 보여준다.)

**브라우저 열기에 실패한 경우(헤드리스/무GUI)에만** `docs/quickstart.html`을 직접 읽어 채팅으로 요약한다:
- **용도 1~2줄**(위와 동일)
- **샘플 파일 표**(파일 → 시나리오) — **기대 판정은 적지 않는다**(기대 결과는 `sample/EXPECTED.md`가 심사·검증용으로 보관):

  | 파일 | 시나리오 |
  |---|---|
  | `sample/input.product.json` | 단건 상품 정합성 검수 데모 |
  | `sample/demo/등록화면캡처.png` | 같은 상품의 등록 화면 캡처 — 실사용 이미지 입력 데모 |
  | `sample/input.pass.json` | 정상 상품 대조군(오탐 확인용) |
  | `sample/input.rescan.json` | 정책(룰) 갱신 → 전 코퍼스 소급 재스캔 데모 |

- **사용법 3경로 요약**: ① 앱에서 "사용해 보기" 클릭 ② 채팅에 프롬프트 복붙(예: `listing-compliance 스킬로 sample/input.product.json을 검수해줘`) ③ CLI(`codex plugin marketplace add …` → `codex plugin add …` → `codex exec`).
- 마지막에 시연 제안 한 줄(상품 이야기로): "번들 샘플 상품 '친환경 에코레더 무스탕 자켓'(690,000원) — 이 상품이 검수에 걸리는지 확인해볼까요?" (위반 내용·기대 판정 예고 금지.)

### 4. 시연 수락 시 — 파일은 사용자가 직접 첨부하게
**바로 번들 샘플을 로드해 돌리지 않는다** — 그러면 사용자에게는 실제 계산과 미리 완성된 결과 호출이 구분되지 않는다. 대신:
1. macOS면 `open -R "<루트>/sample/demo/등록화면캡처.png"`로 시연용 캡처 파일을 Finder에서 선택된 상태로 보여준다(다른 OS면 절대 경로를 채팅에 제시). 이 demo 폴더에는 그 파일 하나만 있다 — 사용자가 헷갈릴 다른 파일을 보여주지 않기 위해서다.
2. 채팅으로 안내(파일명을 그대로 불러서, 굵게 표시 없이): "Finder에 '등록화면캡처.png' 파일을 선택해 두었습니다. 그 파일을 이 채팅에 끌어다 놓고 '검수해줘'라고 해주세요. 첨부하신 이미지에서 표시정보를 추출해 규칙 대조를 수행합니다."
3. 사용자가 파일을 첨부하면 **listing-compliance** 스킬 절차로 그 첨부를 입력 삼아 검수한다(이 스킬은 검수를 직접 수행하지 않는다).
4. 사용자가 첨부 없이 "그냥 돌려줘"라고 하면 그때만 번들 샘플로 진행하고, 응답 서두에 "번들 샘플 시연"임을 밝힌다.

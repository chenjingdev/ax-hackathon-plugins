Listing Compliance

무신사musinsa-listing-complianceCodex 플러그인

상품 표시정보(혼용률·원산지·친환경 표현)를 규칙셋과 시험성적서에 대조해 BLOCK / WARN / PASS로 판정하고, 정정안과 반려 사유서 초안까지 만든다.

30초 시작

  1. 설치: 
    codex plugin marketplace add https://github.com/chenjingdev/ax-hackathon-plugins
codex plugin add musinsa-listing-compliance@ax-hackathon-plugins
  2. Codex 채팅에 붙여넣기 (앱이면 "사용해 보기" 클릭): 
    번들 샘플 sample/input.product.json으로 표시정보 컴플라이언스 검수를 시연해줘
  3. 안감 "캐시미어 30%" 주장이 성적서(아크릴 100%)와 불일치로 걸려 BLOCK 리포트가 나오면 정상.

무엇을 하나

  • 그린워싱 탐지 — 등재 금지표현('에코레더' 등)은 즉시 격발. 미등재 신조어('에코퍼')는 의미로 잡아 사람 확인 후보로 상정.
  • 성적서 대조 — 부위별 혼용률 주장을 시험성적서와 수치 비교 → 정합성 4분류(정상 / 부분커버 / 불일치 / 외래키불명).
  • 소급 재스캔 — 룰이 갱신되면 판매중 코퍼스 전체를 재판정. 자사 PB도 예외 없음.
  • 후속 문서 생성 — 정정안 + 반려 사유서·이의제기 재심사 메모·셀러 응대 초안.

출력

  • 고정 템플릿 리포트(섹션 0~6 + 검증 푸터). 모든 위반에 규칙 ID와 근거 URL이 붙는다.
  • 규칙 ID는 ruleset.json에 등재된 것만 쓴다 — 즉석 창작 금지.

하지 않는 것

  • 자동 게시·차단 없음. BLOCK은 "게시 보류 권고"이고 확정은 사람(MD)이 한다.
  • OCR 없음 — 성적서는 구조화 JSON으로 입력받는다.

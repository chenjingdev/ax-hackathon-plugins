# 샘플 폴더 — 시연 입력 3종 + 기대 결과 문서 1개

이 폴더에는 seeding-screener를 바로 돌려볼 수 있는 **시연 입력 3종**과, 그중 대표 시연의 **기대 결과 문서 1개**가 들어 있다. 아래 표의 "돌려보는 말"을 채팅에 붙여 넣고 해당 파일을 첨부하면 그 자리에서 스크리닝이 실행된다. 각 입력은 스크리너가 하는 일의 다른 면을 보여준다.

## 시연 입력 3종

| 파일 | 시나리오 | 돌려보는 말(파일 첨부 후 붙여넣기) |
|---|---|---|
| `input.kr.json` | 레티놀 세럼 × 후보 4인 — 팔로워 수가 아니라 콘텐츠 결로 후보를 고르는 대표 랭킹 시연 | "이 후보들 스크리닝해줘 — 레티놀 세럼을 누구에게 보내야 할까?" |
| `input.low.json` | 스킨케어와 결이 먼 후보 12인 — 관계없는 후보를 후하게 주지 않는지(과대평가·오탐 방지) 확인하는 대조군 | "이 후보들도 채점해줘 — 결이 먼 후보를 후하게 주지 않는지 보고 싶어" |
| `input.rescan.json` | 채점 기준(룰셋)이 v0.1→v0.2로 바뀐 상황 — 이미 채점한 후보를 소급 재채점하고 순위 변화를 추적하는 재스캔 시연 | "채점 기준이 v0.2로 바뀌었어 — 이 후보들 소급 재채점해줘" |

## 기대 결과 문서

- **`EXPECTED.md`** — `input.kr.json` 시연의 **기대 결과가 박제된 정답지**(심사·검증용)다. 채점 에이전트는 실행 중 이 파일을 열지 않는다 — 미리 답을 베낀 것처럼 보이지 않도록 하는 자기오염 방지 규칙이다. 시연을 돌린 뒤 나온 결과를 이 문서와 눈으로 대조해 채점이 제대로 재현됐는지 확인하는 용도다. `input.low.json`·`input.rescan.json`에는 별도 정답지가 없다.

## 권장 점검 순서

1. **랭킹** — `input.kr.json`으로 대표 시연을 돌려, 후보를 팔로워 수가 아니라 콘텐츠 결로 세우는 기본 동작을 본다.
2. **대조군** — `input.low.json`으로 결이 먼 후보들을 과대평가하지 않는지(오탐 방지) 확인한다.
3. **재채점** — `input.rescan.json`으로 채점 기준 갱신 시 기존 후보를 소급 재채점하고 순위 변화를 추적하는 동작을 본다.

## 캐비엇

후보 콘텐츠(게시물·대본·댓글)는 실존 인물의 verbatim 발언이 아니라, 공개 검증된 한국 뷰티 생태계 아키타입을 기반으로 한 대표 재구성이다(스크래핑 0·날조 방지, 상세 출처는 `EXPECTED.md` 하단 provenance 참조).

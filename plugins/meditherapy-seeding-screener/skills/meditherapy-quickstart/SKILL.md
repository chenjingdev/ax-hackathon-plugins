# meditherapy-seeding-screener 퀵스타트 안내

**읽기 전용 온보딩 스킬이다. 파일을 수정하지 않고, 판정·수치를 즉석에서 창작하지 않는다. 기대 점수·순위를 채팅에 미리 말하지 않는다** — 어떤 후보가 몇 점으로 몇 위가 되는지는 시연 실행이 스스로 보여줘야 하며, 미리 말하면 이후 채점이 답을 베낀 것처럼 보이게 된다(기대 결과 문서는 `sample/EXPECTED.md` — 심사·검증용).

## 절차

### 1) 문서 위치 확인
이 SKILL.md의 **두 단계 상위**(`../..`)가 플러그인 루트다. 거기서 `docs/quickstart.html`이 있는지 확인한다.
(예: 이 파일이 `<루트>/skills/meditherapy-quickstart/SKILL.md`이면 문서는 `<루트>/docs/quickstart.html`.)

### 2) OS 브라우저로 열기 시도
플랫폼에 맞는 명령으로 문서를 연다. 실패하거나 헤드리스 환경이면 조용히 건너뛴다(에러로 중단하지 않음).
- macOS: `open "<루트>/docs/quickstart.html"`
- Windows: `Start-Process "<루트>\docs\quickstart.html"` (또는 `cmd /c start "" "<루트>\docs\quickstart.html"`)
- Linux: `xdg-open "<루트>/docs/quickstart.html"`

### 3) 채팅 출력 — 문서가 열렸으면 짧게, 못 열었을 때만 요약

**2)에서 브라우저 열기에 성공한 경우(기본)**: 문서가 곧 온보딩이다. **HTML에 이미 있는 내용(샘플 표·설치 명령·경제 효과 등)을 채팅에 다시 옮기지 않는다.** 시연 제안은 **파일명이 아니라 후보 이야기로** 한다(사용자에게 `input.kr.json`은 블랙박스다). 채팅 출력은 아래 형식을 넘지 않는다:
1. 소개 문서를 브라우저에 띄웠다는 확인 한 줄
2. 용도 한 줄: 시딩 후보의 공개 콘텐츠를 구매유발 5차원으로 채점해 팔로워 수 대신 콘텐츠 결로 후보를 고르는 도구
3. 시연 메뉴 3종 — 어느 것부터 볼지 사용자가 고르게 한다(**누가 몇 위·몇 점이 되는지, 대조군이 어떻게 나오는지 미리 말하지 않는다** — 결과는 채점이 보여준다. 후보 소개는 문서의 '시연 등장인물'·'시연 시나리오 3종' 섹션에 있다):
   - **① 랭킹**: "레티놀 세럼을 시딩한다면 넷 중 누구에게 보내야 할까요? 문서에 소개된 후보 4인 — 89만 글램 셀럽, 60만 하울 유튜버, 8.2만 리뷰 크리에이터, 8,600 피부과 전문의 — 를 팔로워 수가 아니라 콘텐츠로 채점해 볼까요?"
   - **② 오탐 방지**: "스킨케어와 결이 먼 후보 12인(일상 브이로거·먹방·여행 등)도 채점해, 관계없는 후보를 후하게 주지 않는지 함께 확인해 볼 수 있어요."
   - **③ 재채점**: "채점 기준이 바뀌면 이미 채점한 후보들을 소급 재채점하고 순위가 어떻게 변하는지 추적하는 시연도 있어요."

**브라우저 열기에 실패한 경우(헤드리스/무GUI)에만** `docs/quickstart.html`을 직접 읽어 채팅에 요약한다:
- **용도** 1~2줄: 시딩 후보의 공개 콘텐츠가 구매유발 콘텐츠 결에 얼마나 부합하는지 한국 동료심사 5차원으로 glass-box 스코어링.
- **샘플 표**(파일 → 시나리오) — 기대 점수·순위는 적지 않는다(기대 결과는 `sample/EXPECTED.md`가 심사·검증용으로 보관):

  | 파일 | 시나리오 |
  |---|---|
  | `sample/input.kr.json` | 레티놀 세럼 × 후보 4인 — 대표 스크리닝 데모 |
  | `sample/input.low.json` | 저부합 대조군 12건 — 과대평가(오탐) 방지 확인용 |
  | `sample/input.rescan.json` | 채점 기준 갱신 → 소급 재채점·순위 변화 추적 데모 |

  (각 파일의 상세 설명과 붙여넣을 프롬프트 예시는 `sample/README.md`에 있다.)

- **사용법 3경로**: ① 앱 "사용해 보기" ② 채팅 복붙 프롬프트 ③ CLI 설치+실행.
- 마지막에 시연 제안 한 줄 — 문서가 안 열렸으므로 후보 4인을 채팅에서 직접 소개하며 제안한다: "레티놀 세럼을 시딩한다면 누구에게 보내야 할까요? 89만 글램 셀럽(색조·룩북), 60만 하울 유튜버(협찬 언박싱), 8.2만 리뷰 크리에이터(성분 후기), 8,600 피부과 전문의(성분 기전 설명) — 팔로워 수가 아니라 콘텐츠로 채점해 볼까요?" (기대 결과 예고 금지.)

### 4) 시연 수락 시 — 파일은 사용자가 직접 첨부하게
**바로 번들 샘플을 로드해 돌리지 않는다** — 그러면 사용자에게는 실제 계산과 미리 완성된 결과 호출이 구분되지 않는다. 대신, 사용자가 고른 시나리오에 맞는 파일을 안내한다(① 랭킹 → `input.kr.json`, ② 오탐 방지 → `input.low.json`, ③ 재채점 → `input.rescan.json`):
1. macOS면 `open -R "<루트>/sample/<해당 파일>"`으로 시연용 파일을 Finder에서 선택된 상태로 보여준다(다른 OS면 절대 경로를 채팅에 제시).
2. 채팅으로 안내한다(파일명을 그대로 불러서, 굵게 표시 없이). 시나리오별 문구:
   - **① 랭킹(`input.kr.json`)**: "Finder에 'input.kr.json' 파일을 선택해 두었습니다 — 제품 정보와 후보 4인의 게시물·대본·댓글이 들어 있습니다. 그 파일을 이 채팅에 끌어다 놓고 '스크리닝해줘'라고 해주세요. 첨부하신 콘텐츠를 5차원으로 채점하고 인용을 검증해 랭킹합니다."
   - **② 오탐 방지(`input.low.json`)**: "Finder에 'input.low.json' 파일을 선택해 두었습니다 — 스킨케어와 결이 먼 후보 12인의 콘텐츠가 들어 있습니다. 그 파일을 끌어다 놓고 '이 후보들도 채점해줘'라고 해주세요. 관계없는 후보를 후하게 주지 않는지 함께 봅니다."
   - **③ 재채점(`input.rescan.json`)**: "Finder에 'input.rescan.json' 파일을 선택해 두었습니다 — 이미 채점된 후보들과 갱신된 채점 기준(v0.1→v0.2)이 들어 있습니다. 그 파일을 끌어다 놓고 '소급 재채점해줘'라고 해주세요. 후보들을 새 기준으로 다시 채점하고 순위 변화를 함께 봅니다."
3. 사용자가 파일을 첨부하면 **seeding-screener** 스킬로 그 첨부를 입력 삼아 스크리닝한다(이 스킬은 채점을 직접 수행하지 않는다).
4. 사용자가 첨부 없이 "그냥 돌려줘"라고 하면 그때만 번들 샘플로 진행하고, 응답 서두에 "번들 샘플 시연"임을 밝힌다.
5. 시연 하나를 마친 뒤의 추가 시나리오 제안은 **seeding-screener** 스킬이 담당한다(채점 응답 끝에서 아직 돌리지 않은 샘플 시나리오를 제안).

## 주의
- 이 스킬은 안내만 한다. 스코어링·리포트 생성은 seeding-screener 스킬과 `scoring/cli.py`가 수행한다.
- 기대 점수·순위를 미리 말하거나 없는 수치를 지어내지 않는다 — 결과는 채점 실행이 보여주고, `sample/EXPECTED.md`는 심사·검증용으로만 존재한다.

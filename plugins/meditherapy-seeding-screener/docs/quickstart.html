meditherapy-seeding-screener 표시명: Meditherapy Seeding Screener

인플루언서 시딩 후보의 공개 콘텐츠가 "이 제품을 사게 만드는 결(구매의도 유발 콘텐츠 속성)"에 얼마나 부합하는지를, 한국 동료심사 근거로 가중된 5차원 온톨로지로 정량·정성 스코어링하고 근거를 원문 인용한 glass-box 숏리스트로 산출하는 스크리너.

version 0.2.1 대상 기업 메디테라피 (스킨케어 D2C) 런타임 OpenAI Codex 플러그인 자기완결 외부 네트워크·유료 API 0

1 · 무슨 문제를 푸는가

스킨케어 D2C 브랜드가 신제품(예: 레티놀 세럼)을 시딩할 때, 마케터는 후보 인플루언서 중 "누구에게 제품을 보낼까"를 정해야 한다. 흔한 함정은 팔로워 수로 줄 세우는 것이다 — 팔로워가 큰 계정이 반드시 구매를 유발하는 콘텐츠 결을 가진 것은 아니다.

누가 · 언제 쓰나

  • AX/마케팅 담당자가 시딩 후보 숏리스트를 짤 때 (후보 배열 → 적합도 내림차순 랭킹).
  • 콘텐츠 브리프를 작성할 때 (약한 차원 → 보완 콘텐츠 디렉션 제안).
  • 자사 콘텐츠 사전점검 / 신시장 후보 스크리닝 / 온톨로지 갱신 후 재스캔 시.

정체성 = 예언자가 아니라 정밀 계측기·설명기. 핵심 산출물은 각 차원 점수를 방어하는 원문 verbatim 인용(설명)이며, 점수·랭킹은 정렬용 부산물이다. 마케터가 믿는 것은 숫자가 아니라 "2주간 직접 써보고 말씀드릴게요" 같은 실재하는 근거다.

2 · 폴더 스캐폴딩 맵 (src/)

src/
├─ .codex-plugin/
│  └─ plugin.json          # name · version · description · skills · interface.defaultPrompt
├─ skills/
│  ├─ seeding-screener/SKILL.md        # 메인: 차원별 0~1 부합도 + 원문 인용 산출(에이전트 표면)
│  └─ meditherapy-quickstart/SKILL.md  # 이 온보딩 안내 스킬(읽기 전용)
├─ rules/
│  └─ ontology.json        # 5차원 동결 가중치 + KCI 근거 메타데이터(변경 불가)
├─ scoring/                # 결정론 바닥(순수 python3 stdlib) — 총점은 여기서만 산출
│  ├─ normalize.py · features.py · aggregate.py · report.py · cli.py
│  └─ tests/               # 회귀 테스트(20건) + 자기완결(네트워크 import 0) 검증
├─ sample/                 # input.kr · low · rescan · sagraph + EXPECTED.md(골든 리포트)
└─ docs/
   └─ quickstart.html      # ← 지금 이 문서

판정은 2계층이다. 결정론 바닥(scoring/, 계산 가능 피처, 100% 재현)과 에이전트 표면(Codex가 차원별 0~1 부합도 + 원문 인용). 총점은 오직 코드가 고정 공식으로 산출하고, 에이전트는 전체 점수를 절대 매기지 않는다.

src/sample/에 4개의 검증 픽스처가 있다. 판정은 sample/EXPECTED.md(골든 리포트) 기준이다.

데모 추천: 첫 인상은 sample/input.kr.json이 가장 강하다. 팔로워 1·2위(메가 894k·600k)가 적합도 바닥으로 내려가고, 팔로워 최소(더마 8.6k)가 1위로 역전되는 "팔로워 수의 함정" 4건 역전을 원문 인용으로 방어한다.

4 · 사용하는 3가지 경로

  1. 앱에서 "사용해 보기"

    ChatGPT/Codex 앱에서 이 플러그인의 사용해 보기를 누르면 기본 프롬프트가 뜬다. [0] 퀵스타트 안내, [1] 번들 샘플(sample/input.kr.json) 자가 시연 중 하나로 바로 시작한다.

  2. 채팅에 복붙

    아무 대화창에서 아래를 붙여넣으면 스크리너가 트리거된다.

    번들 샘플 sample/input.kr.json 으로 시딩 스크리닝을 시연해줘.
레티놀 세럼 후보 4인을 구매의도 적합도로 랭킹하고,
각 차원 점수를 원문 인용으로 방어해서 보여줘.

  3. CLI로 설치 + 실행

    Codex CLI에 마켓플레이스를 등록하고 플러그인을 설치한다.

    codex plugin marketplace add https://github.com/chenjingdev/ax-hackathon-plugins
codex plugin add meditherapy-seeding-screener@ax-hackathon-plugins

    설치 후 Codex에서 스크리너를 부르면 된다. 결정론 바닥만 오프라인으로 확인하려면 src/에서:

    python3 -m scoring.cli score --input sample/input.kr.json \
    --ontology rules/ontology.json --deterministic-only \
    --out-json /tmp/kr.floor.json

5 · 기대 출력 미리보기

산출물은 마크다운 리포트(§7.1)와 JSON 로그(§7.3)다. 리포트는 5개 절로 구성된다.

리포트 골격

  1. 후보 랭킹 (purchase_intent_fit · 차원 점수)
  2. 후보별 dossier — 차원 · 점수 · confidence · 원문 인용 · source_field · 논문
  3. 보완 제안 (콘텐츠 디렉션)
  4. 사유 메모 (사람 검토·확정)
  5. 가드 (매출귀인 아님 · reviewed_by=human-required)

판정 체계

  • purchase_intent_fit = round(Σ dimension_score×weight×100), 0~100 정수.
  • 차원 점수 D1~D5 각 0~1 (에이전트 표면).
  • 합산은 고정 공식(코드) — 에이전트가 넘긴 total은 무시.
  • 각 인용은 원문 substring 대조로 grounded 플래그(환각 0 검증).

6 · 제약 · 비목표

절대 하지 않는 것 (가드)

  • 매출 귀인 · 매출 예측 · ROI — 산출하지 않는다.
  • 바이럴 / 도달 / 조회수 예측 — 하지 않는다.
  • 진위(가짜팔로워·봇) 확정 감사 — 하지 않는다(부차 sanity 플래그만, 헤드라인·강등 금지).
  • 모든 출력에 guard 문자열 + reviewed_by="human-required" (자동 발송·확정 0).

지키는 것 (불변식)

  • 근거 실재성(환각 0): 모든 인용은 원문(caption/transcript/comments)에 글자 그대로 실재해야 한다.
  • 가중치 동결: 온톨로지 가중치·블렌드비는 코드 고정값 — 읽기만, 변경 금지.
  • 자기완결: 외부 네트워크·스크래핑·유료 API 0. 영상 자막은 입력으로 받음(스크래핑하지 않음).
  • 총점 분리: 에이전트는 차원별 부합도만, 총점은 코드 고정 공식만.

데모 픽스처의 게시물 텍스트는 플랫폼 접근 제한으로 인한 대표 재구성이며 실제 인물의 verbatim 발언이 아니다 (익명 아키타입 핸들, 스크래핑 0). 표면지표 규모는 공개 분포에 정합.

Listing Factcheck

마이리얼트립myrealtrip-listing-factcheckCodex 플러그인

투어·액티비티 상품의 방문 요일·시기·연령 표기를 명소의 공식 운영사실과 대조해, 지금 팔면 현장에서 노쇼·입장거부 나는 리스팅을 미리 잡는 검수 담당자용 도구.

무엇을 하나

마이리얼트립은 파트너 입점 마켓플레이스다 — 현지 파트너가 수기로 올린 상품을 검수 담당자가 영업일 3일 안에 승인한다(파트너센터 확인). 그런데 검수 항목은 제목·카테고리·태그 같은 메타데이터 중심이고, "월요일 출발 오르세 투어"의 월요일이 실제로 오르세 휴관일인지까지 사람이 명소마다 해외 공식 홈페이지를 뒤져 대조하기는 어렵다. 발행 후에 명소가 휴장 공지를 내면, 통과된 상품이 어긋난 채 계속 팔린다.

  • 3축 대조 — 운영일(휴관 요일·특수휴관) / 시즌(폐쇄 기간) / 연령·신장 규정을 공식 사실 스냅샷과 기계적으로 비교. 근거 URL이 판정마다 붙는다.
  • 소급 재스캔 — 휴장 공지·시즌 경계 같은 변화 1건이 오면 판매중 상품 전체를 다시 판정.
  • 모르면 사람에게 — 확인 안 되는 명소는 차단하지 않고 사람 확인 후보로 올린다.

무슨 결과가 나오나

판정 리포트 한 장: 불일치 목록(주장 vs 공식사실, 근거 URL) + 정정 문구 + 파트너 반려 사유서 초안. 정정은 사람이 읽는 문장으로 나온다 — "매주 월요일 출발" → "화~일 출발(월요일은 오르세 정기 휴관)". 번들 재스캔 데모(판매중 4건)의 기대 결과:

경제적 효과

사고 1건의 비용 구조가 이 도구의 가치다. 현지 사실과 어긋난 상품이 팔리면 당일 노쇼·입장거부 → 전액 환불 + CS 응대 + 파트너 분쟁 + 평점·재구매 하락이 한꺼번에 발생하고, 지금 구조에서는 터진 뒤에야 발견된다.

  • 사고 사전 차단 — 발행 당시 맞았어도 세계가 바뀌면 어긋난다. 시즌 경계·휴장 공지마다 판매중 전체를 자동 재점검해, 환불·CS·분쟁 비용이 발생하기 전에 내린다.
  • 검수 공수 축소 — 명소 공식 홈페이지를 사람이 뒤져 대조하는 축을 자동화. 검수자는 걸린 건과 사람 확인 후보만 본다.
  • 리드타임 단축 — 영업일 3일 검수 게이트에서 사실 대조 축이 즉시 처리되면 상품 개시가 빨라진다 = 판매 기회.
산정 예시 — 리스팅 1건의 명소 사실 대조를 15분으로 잡으면(가정) 판매중 1,000건 재점검 = 250시간 ≈ 31인일. 사람은 세계상태가 바뀔 때마다 이걸 반복할 수 없지만, 도구는 변화 1건마다 무상으로 반복한다.

마이리얼트립의 특정 노쇼 사고를 다룬 1차 출처는 사전조사에서 확보되지 않았다(정직한 한계). 위 산정은 확인된 검수 구조(3일 게이트·메타데이터 항목)에 보수적 가정을 붙인 추정이다.

선 긋기

자동 게시·발송은 없다. BLOCK은 "개시 보류 권고"이고 확정은 검수 담당자가 한다. 법률 심사·여정 설계가 아니고, 사실은 근거 URL이 붙은 동결 스냅샷에서만 온다.

myrealtrip-listing-factcheck

표시명 Listing Factcheck · 입점 T&A 상품 ↔ 명소 공식 운영사실 정합 상시 검수·소급 재스캔

입점 T&A 상품(투어·액티비티·입장권)의 방문 요일·시기·연령이 명소 공식 운영사실과 어긋나 “지금 팔면 현장에서 노쇼·입장거부 나는” 리스팅을, 등록·수정·세계상태 변화(휴장 공지·시즌 경계)마다 3축 결정론 대조로 사전 차단하는 Codex 플러그인.

version 0.2.1 대상 기업 마이리얼트립 런타임 외부 API·키 0 3축 운영일 · 시즌 · 연령

1무슨 문제를 푸는가

마이리얼트립은 자체 기획이 아니라 파트너 입점 마켓플레이스다. 현지 파트너가 투어·액티비티·티켓을 올리면 검수 담당자가 가이드라인 대조로 개시 여부를 승인한다(human-in-the-loop, 영업일 3일 검수 게이트).

그 수기 검수는 제목·카테고리·태그·대표사진 같은 메타데이터 항목을 다루지만, 리스팅이 주장하는 방문 요일·시기·연령이 명소의 실제 현지 운영사실(휴관 요일·시즌 폐쇄·연령/신장 규정)과 맞는지까지는 구조적으로 닿기 어렵다(추정). 그래서 발행 시점엔 통과됐던 리스팅이, 세계가 바뀌면(휴장 공지·시즌 경계 도래) 스스로 어긋난 채 팔리다 현장 CS로 터진다.

누가·언제 — 검수 담당자가, 상품 등록·수정 시점뿐 아니라 명소 세계상태 변화가 있을 때마다. 이 에이전트는 그 시점에 판매중 코퍼스를 소급 재스캔해 걸리는 리스팅을 공식 근거(URL)와 함께 끌어올린다.

4사용법 — 3경로

1앱에서 “사용해 보기”

ChatGPT/Codex 앱에서 플러그인을 열고 “사용해 보기”를 누르면 첫 기본 프롬프트가 이 퀵스타트를 띄운다.

퀵스타트: 이 플러그인의 용도·구조·샘플 시연 방법을 안내해줘

2채팅에 프롬프트 복붙

바로 킬러 데모를 돌리려면 아래를 채팅에 붙여넣는다.

번들 샘플 sample/input.rescan.json 코퍼스를 재스캔 검수해서,
시즌 경계 변화로 지금 팔면 현장에서 걸리는 판매중 리스팅을 소급 적발해줘

3CLI 설치 + 실행

Codex CLI에 마켓플레이스를 등록하고 플러그인을 설치한다.

codex plugin marketplace add https://github.com/chenjingdev/ax-hackathon-plugins
codex plugin add myrealtrip-listing-factcheck@ax-hackathon-plugins

설치 후, 검수 대상 입력 없이 실행하면 sample/의 대표 샘플을 “번들 샘플 시연”으로 자동 로드해 보여준다.

5기대 출력 미리보기

검수 스킬은 아래 마크다운 리포트 골격(섹션 번호·순서 고정)으로 응답한다.

## 현지 운영사실 정합성 검수 리포트
   - 모드 · 상품 · 판정: BLOCK n · WARN m
1) 결정론 바닥 — 탐지된 불일치   # 표: #·규칙 ID·심각도·명소·주장 vs 공식사실·근거 URL
2) 에이전트 표면 — 상정          # 명소불명 review_candidate · SC-02
3) 소급 재스캔 결과              # 사실 델타 · newly_broken · passed (재스캔 모드만)
4) 정정안                        # before → after (공식 사실 기준 재작성)
5) 출력단 초안                   # 파트너 반려 사유서 / 고객 안내문
6) 결론

판정 체계

  • BLOCK = “개시 보류 권고”(자동 차단 아님). 위반 BLOCK ≥ 1이면 리스팅 판정 BLOCK.
  • WARN = 주의. BLOCK = 0이고 WARN ≥ 1일 때.
  • PASS = 위반 0. 깨끗한 상품은 막지 않는다.

규칙 ID (3축 + 전처리)

축규칙 ID탐지 대상
R1 운영일 정합OD-01 · OD-02 · OD-03방문 요일/날짜/시각 vs 정기 휴관·특수 휴관·운영시간
R2 시즌·기후SC-01 · SC-02판매 시기 vs 시즌 전면 폐쇄(SC-01 결정론) / 계절 키워드(SC-02 자문)
R3 연령·자격AG-01 · AG-02아동/유아 요금 연령·신장 vs 공식 규정
전처리PP-01 · PP-02명소·요일/시기 누락 또는 날짜 모순 → 사람에게 되돌림

6제약 · 비목표

  • 자동 게시·발송 0 — 모든 정정·초안은 제안이며 reviewed_by="human-inspector-required".
  • 법률 심사 아님 — 검사 대상은 조문이 아니라 명소 공식 운영사실이다.
  • 여정 설계 아님 — 검사 단위는 여정 최적화가 아니라 단품 리스팅 1건.
  • 런타임 외부 네트워크·API 키 0 — 동결 facts.json 스냅샷만 대조(라이브 API 아님).
  • 블랙박스 예측 아님 — glass-box, 모든 판정에 source_url이 붙는다.
  • 결정론 바닥(3축 사실 대조)은 하드게이트, 에이전트 표면(명소 퍼지매칭·명소불명 상정·SC-02)은 판단성 자문이라 하드 BLOCK을 강제하지 않는다.

Listing Factcheck

마이리얼트립myrealtrip-listing-factcheckCodex 플러그인

투어·액티비티 상품의 방문 요일·시기·연령 표기를 명소 공식 운영사실과 대조해, 지금 팔면 현장에서 노쇼·입장거부 나는 리스팅을 잡는다.

30초 시작

  1. 설치: 
    codex plugin marketplace add https://github.com/chenjingdev/ax-hackathon-plugins
codex plugin add myrealtrip-listing-factcheck@ax-hackathon-plugins
  2. Codex 채팅에 붙여넣기 (앱이면 "사용해 보기" 클릭): 
    번들 샘플 sample/input.rescan.json 코퍼스를 재스캔 검수해줘
  3. 판매중 4건 중 3건이 BLOCK(휴관일 겹침·우기 폐쇄·연령 미달), 1건이 PASS로 나오면 정상.

무엇을 하나

  • 3축 대조 — 운영일(휴관 요일·특수휴관) / 시즌(폐쇄 기간) / 연령·신장 규정을 facts.json의 공식 사실과 결정론 비교.
  • 소급 재스캔 — 휴장 공지·시즌 경계 같은 세계상태 변화 1건으로 판매중 코퍼스 전체 재판정.
  • 명소불명 상정 — 매칭 안 되는 명소는 차단하지 않고 사람 확인 후보로 올린다.
  • 후속 문서 생성 — 정정 표기 + 파트너 반려 사유서·고객 안내문 초안.

폴더 구조

myrealtrip-listing-factcheck/
├── .codex-plugin/plugin.json    매니페스트

│   └── myrealtrip-quickstart/   이 문서를 여는 온보딩 스킬
├── rules/
│   ├── ruleset.json             규칙 9개(OD·SC·AG·PP)·임계
│   ├── facts.json               명소 공식 운영사실 동결 스냅샷(근거 URL 포함)
│   └── sources.md               출처 목록
├── sample/                      데모 입력 2종 + 기대 출력
└── docs/quickstart.html         이 문서

출력

  • 고정 6섹션 리포트 + JSON 로그. 모든 위반에 규칙 ID와 공식 근거 URL이 붙는다.
  • 정정안은 기계 표현이 아니라 사람이 읽는 문장으로 — 예: "매주 월요일 출발" → "화~일 출발(월요일은 오르세 정기 휴관)".

하지 않는 것

  • 자동 게시·발송 없음. BLOCK은 "개시 보류 권고"이고 확정은 검수 담당자가 한다.
  • 법률 심사·여정 설계 아님. 라이브 API 아님 — 사실은 동결 스냅샷(facts.json)에서만 온다.
